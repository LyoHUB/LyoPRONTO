# Example Outputs

This directory contains output files generated by example scripts.

## Purpose

This directory stores example outputs to demonstrate what LyoPRONTO produces. These files serve as:
- **Documentation**: Shows users what to expect
- **Validation**: Reference outputs for comparison
- **Examples**: Templates for result formatting

## Files

### CSV Files
Generated CSV files with simulation results in web interface format:
- Semicolon-separated values (`;`)
- Standard columns: Time, Temperatures, Pressure, Flux, % Dried
- Timestamped filenames: `lyopronto_primary_drying_MMM_DD_YYYY_HH_MM_SS.csv`

### Plot Files
Generated visualization files:
- `primary_drying_results.png`: 4-panel plot showing temperature, flux, drying progress, and pressure profiles
- High resolution (300 DPI) for publication quality

## Generation

These files are generated by running:
```bash
python examples/example_web_interface.py
```

## Version Control

**Policy**: Example outputs are tracked in git to provide reference examples, but generated outputs during development should be .gitignored.

**What's tracked**:
- ✅ Reference example from original web interface validation
- ✅ Key example outputs for documentation

**What's ignored**:
- ❌ Newly generated outputs (add to .gitignore pattern)
- ❌ Temporary or test runs
- ❌ Large files (>5 MB)

## Usage

### For Users
These files show what you should expect when running LyoPRONTO examples. Compare your results with these reference outputs.

### For Developers
When updating example scripts:
1. Run the example
2. Verify output matches expected format
3. If intentional changes, update reference files
4. Document changes in commit message

## Cleanup

To remove generated files and start fresh:
```bash
rm examples/outputs/lyopronto_primary_drying_*.csv
rm examples/outputs/primary_drying_results.png
```

Then re-run the example to regenerate.
